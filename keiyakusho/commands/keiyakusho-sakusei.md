ラビーネット（全日本不動産協会）の統一書式で不動産契約書を作成する。

1. keiyakushoスキルを読み込む（SKILL.mdとreferences/書式インデックス.md）。

2. 引数から契約種類を取得する。引数がない場合はAskUserQuestionで以下を順に確認する:
   - 取引形態: 売買 / 賃貸借 / 媒介契約 / 覚書
   - 物件タイプ: 土地 / 土地建物 / マンション / 借地権付建物 / 駐車場 / 事業用
   - 契約形態の詳細（売買なら一般仲介/消費者契約/売主宅建業者、賃貸借なら保証形態・借家形態）

3. `references/書式インデックス.md` を参照して、適切なテンプレートファイルを特定する。

4. AskUserQuestionで調査データの有無を確認する:
   - 「物件の調査資料はありますか？（PDFファイル、物件フォルダなど）」
   - 選択肢: 「資料あり（ファイルを渡す）」/「資料なし」

5. **資料ありの場合** → ステップ6へ進む:
   利用者から資料を受け取り、`references/調査結果_契約書マッピング.md` を参照してデータを読み込む。不足情報があればヒアリングで補完する。

6. **資料なしの場合** → チェックリストを渡して終了:
   口頭ヒアリングだけで契約書を埋めようとしてはいけない。以下を実行する:
   a. fudosan-chosaスキルのSKILL.mdを読み込む
   b. ステップ2で特定した物件タイプに合った調査チェックリスト（Excelファイル）を実際に生成してユーザーに渡す
   c. 「この調査チェックリストの資料が揃ったら、もう一度 /keiyakusho-sakusei で契約書を作りましょう」と案内する
   d. **ここで処理を終了する。ステップ7以降には絶対に進まない。**

7. 当事者情報・取引条件など、調査データで埋まらない不足情報をヒアリングする。

8. `scripts/fill_template.py` と `scripts/utils.py` を使ってExcelテンプレートに自動入力する。

9. 記入漏れや金額の整合性を確認する。

10. 完成したファイルを物件フォルダ（あれば）の `06_契約書類/` に保存するか、ワークスペースに保存してリンクを返す。
