allowed-tools: Read, Write, Bash, Glob, Grep

---

新しい物件フォルダを作成する。

1. 引数から物件名と住所を取得する。引数がない場合はAskUserQuestionで聞く。

   **重要: AskUserQuestionの選択肢の例は必ず大分市・大分県内の地名を使うこと。東京・大阪・名古屋など他県の地名は絶対に使わない。**

   質問1: 物件名と住所を聞く
   - 選択肢の例:
     - 「荷揚町中古戸建 / 大分市荷揚町2-31」
     - 「明野東パークハイツ / 大分市明野東1丁目」
     - 「わさだタウン横 土地 / 大分市大字玉沢」
   - 「その他」でユーザーが自由入力できるようにする

   質問2: 物件タイプ
   - 土地 / 土地建物 / マンション（区分所有） / 借地権付建物 / 事業用 / 駐車場

2. ユーザーのワークスペースフォルダに以下の構成でフォルダを作成する:

```
{物件名}_{住所}/
├── 01_事前調査/
│   ├── おおいたマップ情報.md
│   └── 登記情報メモ.md
├── 02_法務局/
│   └── 取得資料リスト.md
├── 03_現地調査/
│   └── 現地調査メモ.md
├── 04_役所調査/
│   ├── 都市計画課.md
│   ├── 道路建設課.md
│   ├── 開発建築指導課.md
│   └── その他.md
├── 05_インフラ/
│   ├── 上水道.md
│   ├── 下水道.md
│   └── ガス.md
├── 06_契約書類/
└── 調査チェックリスト.md
```

3. 各.mdファイルには、物件タイプに応じたテンプレート内容を入れる。fudosan-chosaスキルの`references/不動産調査マニュアル.md`と`references/大分市_調査ガイド.md`を読み込んで、物件タイプに合わせたチェック項目を生成する。

4. 調査チェックリスト.mdには、マークダウンのチェックボックス形式（`- [ ]`）で全調査項目を列挙する。大分市の窓口情報（何課・何階）も記載する。

5. 完了したらフォルダのリンクを返す。
